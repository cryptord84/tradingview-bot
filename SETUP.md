# TradingView SOL Trading Bot - Setup Guide

## Requirements
- macOS Sequoia (or any macOS/Linux)
- Python 3.11+
- TradingView account (Pro/Pro+ for webhook alerts)
- Anthropic API key
- Solana wallet (Phantom) with SOL for trading
- Optional: Telegram bot, NewsAPI key

## Step 1: Clone & Create Virtual Environment

```bash
cd ~/Projects/tradingview-bot
python3 -m venv venv
source venv/bin/activate
pip install -r requirements.txt
```

## Step 2: Get API Keys

### Claude AI Decision Layer (choose one)

**Option A - Claude Code CLI (default, no API key needed):**
1. Install Claude Code: `npm install -g @anthropic-ai/claude-code`
2. Run `claude` once in your terminal to authenticate
3. In config.yaml, set `claude.mode: "cli"` (this is the default)
4. The bot calls `claude --print` for each decision, using your existing auth

**Option B - Anthropic API (faster, costs per token):**
1. Go to https://console.anthropic.com
2. Create an API key
3. In config.yaml, set `claude.mode: "api"` and `claude.api_key: "sk-ant-..."`

### TradingView Webhook
1. Need TradingView Pro/Pro+ for webhook alerts
2. Choose a webhook secret (any random string)
3. Your webhook URL will be: `http://YOUR_IP:8000/webhook`
4. For external access, use ngrok: `ngrok http 8000`

### Solana RPC (recommended: paid RPC for reliability)
- **Free**: `https://api.mainnet-beta.solana.com` (rate limited)
- **Helius**: https://helius.dev (free tier available)
- **Alchemy**: https://alchemy.com
- **QuickNode**: https://quicknode.com

### Telegram Bot (optional)
1. Message @BotFather on Telegram
2. `/newbot` → follow prompts → get bot token
3. Message your bot, then get chat_id via: `https://api.telegram.org/bot<TOKEN>/getUpdates`

### NewsAPI (optional)
1. Go to https://newsapi.org
2. Sign up for free tier (100 requests/day)

## Step 3: Configure

```bash
cp config.yaml.example config.yaml
cp .env.example .env
```

Edit `config.yaml` with your API keys and preferences.

## Step 4: Encrypt Your Wallet Private Key

**CRITICAL: Use a dedicated trading wallet with limited funds. Never use your main wallet.**

1. Export private key from Phantom:
   - Settings → Security & Privacy → Export Private Key
2. Encrypt it:

```bash
source venv/bin/activate
python -m app.utils.encrypt_key
```

3. Paste the encrypted output into `config.yaml` → `wallet.encrypted_private_key`
4. Set your password in `.env`:
   ```
   WALLET_ENCRYPTION_PASSWORD=your_password_here
   ```

## Step 5: Set Up TradingView

1. Open TradingView, add the strategy:
   - Pine Editor → paste contents of `strategy.pine` → Add to Chart
2. Configure the strategy settings (adjust indicators to your preference)
3. Set the webhook secret in the strategy settings to match `config.yaml`
4. Create an alert:
   - Right-click chart → Add Alert
   - Condition: "SOL Confluence Pro"
   - Webhook URL: `http://YOUR_PUBLIC_URL/webhook`
   - The alert message is auto-generated by the Pine Script

## Step 6: Run the Bot

```bash
source venv/bin/activate
python -m uvicorn main:app --host 0.0.0.0 --port 8000
```

For development with auto-reload:
```bash
python -m uvicorn main:app --host 0.0.0.0 --port 8000 --reload
```

## Step 7: Access Dashboard

Open http://localhost:8000 in your browser.

API docs at http://localhost:8000/docs.

## Step 8: Expose Webhook (for TradingView)

TradingView needs a public URL. Options:

### ngrok (easiest)
```bash
brew install ngrok
ngrok http 8000
```
Use the `https://xxxx.ngrok.io/webhook` URL in TradingView alerts.

### Cloudflare Tunnel (free, more stable)
```bash
brew install cloudflared
cloudflared tunnel --url http://localhost:8000
```

## Resource Usage

Tested on MacBook Pro i7 16GB:
- Idle: ~50MB RAM

- No Docker, no heavy ML libraries
- SQLite for storage (no external DB)

# Security Best Practices

1. **Never commit secrets**: `.gitignore` already excludes `config.yaml`, `.env`, and `data/`
2. **Use a dedicated trading wallet**: Fund it with only what you're willing to risk
3. **Hardware wallet**: For large amounts, use a Ledger/Trezor and only keep trading funds in the hot wallet
4. **Encrypt private key**: Always use the encryption tool, never store raw keys
5. **Restrict webhook access**: Use ngrok's IP allowlisting or Cloudflare Access
6. **Monitor balance**: The bot auto-shuts down below the low-balance threshold
7. **Start small**: Begin with minimal position sizes and increase gradually
8. **Review Claude decisions**: Check Telegram notifications and dashboard regularly
9. **Keep dependencies updated**: `pip install --upgrade -r requirements.txt`
10. **Backup trade data**: CSV backups are automatic; also backup `data/trades.db`
11. **RPC security**: Use a private RPC endpoint, not the public one, for production
12. **Rate limiting**: Built-in, but also consider firewall rules
13. **Never expose port 8000 directly**: Always use ngrok/cloudflare tunnel
